# S-DATA 数据分析与质控

## 工作定位

| 项 | 内容 |
|---|------|
| **前置条件** | S3写作蓝图确认后（可与S3起草并行启动） |
| **门禁类型** | **软确认**（`status: reported`，产出汇报即可） |
| **阶段定位** | **可选阶段** — 仅实证/量化类论文触发 |
| **后置动作** | 更新 state.json（S-DATA→reported）+ 追加 LOG.md |

## 触发判定（重要）

### 自动触发（paper-os 总控调度时复核）

论文类型 ∈ {**实证研究、量化研究、问卷调查、统计分析、实验研究**} → **触发 S-DATA**

论文类型 ∈ {规范分析、纯理论研究、案例分析、文献综述} → **默认跳过**，输出提示：

`⚠️ 本论文类型（[具体类型]）建议跳过数据分析阶段。如需强制启动 S-DATA，请明示。`

### 手动触发（用户直接说"数据分析"等触发词）

不论文文类型，直接进入路径选择。但若论文类型属于跳过集合，先输出上述提示并请用户确认。

> **注**：paper-os 总控在项目初始化时已做论文类型判定并写入 `state.json`；本技能被调度时**复核**该判定，发现类型不匹配时提示而非静默执行。

## 两条路径（二选一或先后执行，不强求都跑）

| 路径 | 工具 | 适用研究者 | 解决问题 |
|------|------|-----------|----------|
| **A. 智谱清言 GLM4 零代码分析** | GLM4 数据分析功能 | 无编程基础研究者 | 描述统计/可视化/建模/对比/进退步分析 |
| **B. Codex 脏数据 6 步核查** | Codex + 工作目录隔离 | 有编程基础 / 需严谨质控 | 字段命名不一/单位混乱/计算口径模糊 |

**路径选择建议**：
- 数据干净、只需常规统计分析 → **路径 A**
- 数据存在命名混乱、单位不一、口径模糊等质控问题 → **先 B 后 A**（先洗干净再分析）
- 两条路径不强求都跑，按实际问题选择

> 路径 A 详细流程见 [`references/data-analysis-gl.md`](references/data-analysis-gl.md)
> 路径 B 详细流程见 [`references/data-qa-codex.md`](references/data-qa-codex.md)

## 三条安全红线（硬失败级）

> 源自脏数据核查场景，违反任意一条即**硬失败**，立即终止并报告用户。

| 红线 | 内容 | 违反后果 |
|------|------|----------|
| **1. 只读优先** | 第一阶段仅扫描盘点，**禁止直接修改源文件** | 硬失败 |
| **2. 仅操作测试副本** | 所有数据操作必须在 `materials/_data_workspace/` 测试副本内进行，源文件路径只读 | 硬失败 |
| **3. 禁止覆盖源文件** | 质检清单/问题清单/修复版**全部另存为新文件**，修复版强制加 `_修复版` 后缀 | 硬失败 |

---

## 技术保障（红线从 prompt 约束升级为强制执行步骤）

红线不能只靠 prompt 提示，必须用文件系统机制强制执行：

1. **工作目录隔离**：Codex / GLM4 工作目录强制设为 `materials/_data_workspace/`；源 CSV 复制进副本，源文件路径对工具只读
2. **修复版命名强制**：所有修复版文件名强制带 `_修复版` 后缀（如 `成绩单_修复版.csv`）
3. **mtime 自检（强制步骤）**：
   - 执行前：`ls -la <源文件>` 记录源文件 mtime
   - 执行后：再次 `ls -la <源文件>` 确认 mtime **未变化**
   - **mtime 变化 = 硬失败**（说明源文件被改写），立即报告用户并中止

## GLM4 结果进论文的门禁（关键）

> **核心原则**：AI 是帮手不是枪手。GLM4 零代码分析可能产生错误统计或幻觉数据，是"幽灵文献的数据等价物"。

**门禁规则**：

S-DATA 产出的**任何**进入论文正文的统计数字 / 图表 / 模型结论，**必须经 S6 事实核查复核通过后方可引用**。

- S6 核查通道表已新增 **S-DATA 产出类型**行（数据/统计 → 原始数据源核对）
- S3 起草引用 S-DATA 产物时，脚注标注 `待 S6 核查`
- 理由：违反 paper-os 核心原则 4「幽灵文献零容忍」的数据等价物版

**禁止行为**：
- 直接把 GLM4 输出的统计数字写入论文正文不加标注
- 把 GLM4 生成的图表直接作为论文插图而不经 S6 复核
- 引用未经 S6 复核的模型结论作为论文核心论据

## 输出约定

保存到：`outputs/S-DATA-数据分析/`

| 产物 | 来源路径 | 说明 |
|------|---------|------|
| `数据分析报告.md` | 路径 A | GLM4 分析结果汇总（图表+数据表+解读） |
| `脏数据核查清单.md` | 路径 B 第 3 步 | 《自检清单》每项含 检查内容/合格标准/严重等级 |
| `问题清单.md` | 路径 B 第 4 步 | 全量问题逐项含 严重程度/位置/原文/建议修复 |
| `*_修复版.csv` | 路径 B 第 5 步 | 修复后数据（**不改源**，加 `_修复版` 后缀） |
| `检查日志.md` | 路径 B 第 5 步 | 修复对照清单 + 操作记录 |

### 产物 frontmatter

```yaml
---
step_id: S-DATA
status: reported
key_conclusions:
  - 数据分析类型：描述统计/可视化/建模/质控
  - 核心发现：（1 条，最多 3 条）
  - 源文件完整性：源文件 mtime 未变 / _修复版已生成
depends_on:
  - S3
deliverables:
  - 产出文件路径列表
---
```
