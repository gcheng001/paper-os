## 执行流程

### Step 1：文献筛选（优先级排序）

按来源可信度排序：

| 优先级 | 来源 | 说明 |
|--------|------|------|
| 最高 | 顶级期刊/顶会 | 本领域公认的权威出版物 |
| 高 | 领域主流期刊 | 被广泛引用的专业期刊 |
| 中 | 学位论文（博士>硕士） | 综述性强，适合入门 |
| 低 | 工作论文/预印本 | 未经同行评议，需谨慎 |

**法律文献补充**：通过元典检索规范性材料——
- `yuandian_law_vector_search`：法条语义检索
- `yuandian_rh_fg_search`：法规检索
- `yuandian_rh_qwal_search`：权威/典型案例裁判要旨

### Step 2：卡尔非线性阅读顺序

**不要从头到尾线性阅读。** 来源：明尼苏达大学卡尔教授（已发表300+篇论文）。

**核心原则：你可以随时停止阅读一篇文章，甚至将其丢弃。** 读文献要遵守经济规律，不要试图打捞沉没成本。在不看好或不感兴趣的文献上投入更多时间是不明智的。

```
① 标题 / 关键词 / 摘要
   → 判断是否值得继续（30秒决策）

② 结论
   → 看研究发现了什么

③ 图表（关键决策点）——注意：不要先读引言！
   → 图表能快速传达核心信息
   → 看不懂图表 → 该文献可能超出当前知识水平

④ 引言
   → 了解研究背景和动机

⑤ 结果与讨论
   → 详细了解发现及其意义
   → 对研究结果的讨论要解答读者可能存在的疑问，从实验结果中挖掘全部蛛丝马迹

⑥ 方法/实验（仅高价值文献）
   → 只有你需要复现或借鉴方法时才深入
   → 这是最困难的部分，速度要慢下来，甚至需要花时间尝试复现
```

### Step 3：文献笔记法

每篇文献记录以下要素（生成卡片存 `outputs/S2-文献/文献笔记/`）：

```markdown
# [文献标题]
- **作者/年份**：
- **来源**：（期刊/会议/出版社）
- **核心观点**：（一句话）
- **研究方法**：
- **关键发现**：
- **与本研究的关系**：（支持/反对/补充/无关）
- **可引用点**：
- **疑问/批判**：
```

### Step 4：AI辅助论文解读（读不懂时使用）

> **AI是帮手，不是枪手。** 不要把AI总结当阅读心得提交，不要让AI替你读论文。

#### 4a. "人话说论文"提示词

```
用通俗易懂的语言给我讲讲作者的研究设计、过程、结果，
然后帮我找找这个研究有什么不合理的地方。
```

**为什么必须加后半句？**
- AI通常顺着论文思路一味夸赞（训练时的"迎合"倾向）
- "找不合理的地方"强制AI运用批判式思维审视论文
- 避免把烂文章也夸成鲜花

**模型选择建议**：选择幻觉足够低的模型。Gemini系列在幻觉排行榜上表现较好。模型温度设为Medium（中等），太高缺乏客观性。

> 详细示例见 `references/人话说论文示例.md`（以Transformer论文为案例的完整解读）。

#### 4b. 数学公式解读

接续"人话说论文"对话，继续提问：

```
我对论文中的数学公式很感兴趣，请你把其中出现的所有数学公式的含义，
也用刚才那种（通俗易懂的）风格给我解释一遍。
```

> AI会将全部公式耐心梳理，用比喻和真实场景帮助理解。

#### 4c. 可视化公式解释

```
对第一个公式，我理解起来还是不够清楚，你能用SVG或者Mermaid来为我进行可视化讲解吗？
```

> AI会生成Mermaid代码→粘贴到mermaid.live→可缩放示意图，精细到足够放入正式报告。

#### 4d. 交互式教学网页（高级）

```
目标与受众：
- 面向：大二数据科学系学生（注意力易分散，喜欢动效与互动）
- 任务：围绕用户输入的内容做科普式中文讲解，兼顾深度与易懂
- 交付：一个压缩包（含index.html和必要资源），用户下载后直接双击即可浏览

内容与表现：
1. 知识要点：资料准确、示例鲜活
2. 视觉原则：多用图形、动态图表；避免连续长段文字
3. 结构：引入与悬念→核心概念动画演示→交互实验区→拓展与发展史→总结与延伸阅读
4. 技术：纯前端实现，即开即用，确保Chrome/Edge/Safari正常运行
```

#### 4e. 层级递进式理解（如果AI解读后仍不懂）

1. 补充基础知识：读论文参考文献列表中的基础文献
2. 找教学资源：PPT、视频课程、博客
3. 最后才"死磕"原文

### Step 5：生成文献矩阵

汇总所有文献笔记，输出文献矩阵表：

```markdown
| 文献 | 核心观点 | 方法 | 与本研究关系 | 优先级 |
|------|---------|------|-------------|--------|
| 张三(2024) | ... | 实证 | 支持H1 | 高 |
| Li(2023) | ... | 理论 | 反对H2 | 高 |
```

### Elicit 系统化文献回顾流程

1. 输入研究问题 → Elicit自动判断是否足够明确，给出细化建议
2. 文献获取 → 从Elicit库搜索最多500篇相关论文
3. 定义过滤条件 → Elicit自动建议筛选维度（AI干预、实证数据、样本量等）
4. 自动筛选 → 对全部文献打分（满分5分），按分数排序
5. 信息抽取 → 自动抽取研究设计、参与者特征、关键发现等
6. 生成报告 → 系统化文献回顾报告，含分项数据汇总、证据支持

> **警告**：AI生成的文献回顾必须逐条核对引用内容，不要直接发给导师或投稿。
