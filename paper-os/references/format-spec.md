# 论文终稿格式规范（paper-os）

> paper-os 论文终稿输出的**字体版面**与**引注体例**统一规范。
> 字体版面已固化为 md2word 预设 `paper-os.yaml`，`/导出Word` 默认套用。
> 引注体例蒸馏自《中文引注体例》（完整原文由用户提供，存于项目 `materials/`）。

---

## 一、字体版面规范

### 1.1 字号换算表（中文字号 ↔ pt）

| 字号 | pt | 字号 | pt |
|------|----|------|----|
| 初号 | 42 | 小三 | 15 |
| 小初 | 36 | 四号 | 14 |
| 一号 | 26 | 小四 | 12 |
| 小一 | 24 | 五号 | 10.5 |
| **二号** | **22** | 小五 | 9 |
| 小二 | 18 | 六号 | 7.5 |
| 三号 | 16 | | |

### 1.2 核心格式（用户指定，硬性）

| 元素 | 字体 | 字号 | 其他 |
|------|------|------|------|
| **论文题目（H1）** | **宋体** | **二号（22pt）** | 加粗、居中 |
| **正文** | **仿宋** | **四号（14pt）** | 首行缩进2字符、两端对齐 |

### 1.3 标题层级（已确认）

> 论文除大题目外，还有章、节、小节标题。H2/H3 采用黑体分级以区别正文仿宋，**已与用户确认采用**（2026-06-26）。投稿时若目标期刊/学校另有要求，按第四节调整入口修改。

| 层级 | 对应 | 字体 | 字号 | 说明 |
|------|------|------|------|------|
| H1 | 论文题目 | 宋体 | 二号(22) | 居中加粗 |
| H2 | 一级标题（一、二、） | 黑体 | 三号(16) | 左对齐加粗 |
| H3 | 二级标题（（一）（二）） | 黑体 | 四号(14) | 左对齐加粗 |
| H4 | 三级标题（1. 2.） | 仿宋 | 四号(14) | 加粗（与正文同字体，加粗区分） |

> 备选：若目标期刊要求章节标题统一宋体，把 H2/H3 的 `font` 改为「宋体」即可。

### 1.4 段落与页面（已确认）

| 项 | 值 | 备注 |
|----|----|------|
| 行距 | 1.5 倍 | 已确认；学位论文常要求「固定值20磅」或「2倍」，届时按目标要求改 `paragraph.line_spacing` |
| 首行缩进 | 2字符（28pt） | 四号字 × 2 |
| 对齐 | 两端对齐 | — |
| 纸张 | A4（21×29.7cm） | — |
| 页边距 | 上下2.54 / 左右3.18 cm | 论文常用，可调 |
| 页码 | 页脚居中，五号 Times New Roman | 格式「1」 |

### 1.5 摘要、关键词、参考文献等特殊段落

md2word 按 Markdown 结构渲染，以下为写作约定（不强制字号，导出后可单独调）：

- **摘要**「摘要：」二字加粗（黑体小四），后接仿宋小四；摘要标题「摘要」居中黑体三号。
- **关键词**「关键词：」加粗，词与词用分号隔开，仿宋小四。
- **参考文献**标题黑体三号居中；条目仿宋五号（与脚注体例一致）。
- **脚注**：Word 脚注默认小五/五号宋体，按目标要求统一。

> 这些元素如需在预设层面固化字号，扩展 `paper-os.yaml` 对应字段即可。

## 二、引注体例（脚注/参考文献规范）

> 论文终稿的注释与文献引用必须遵循本节。S6 事实核查环节据本节核对引用准确性。

### 2.1 通则

**引注符号位置**
- 引用全句：符号置于句号、问号**之后**。
- 引用句中部分内容：符号置于该部分**之后**、其他标点之前。
- 直接引用字词：符号**紧接引号**，置于其他标点之前。

**引领词**
- 「参见」：非原文直引时加；直接引用**不加**「参见」及任何引领词。
- 「转引自」：未核实原初文献、转引他人时用。
- 「载」：文献来源于期刊、报纸、网络、文集时，来源前标注。

**同一注释含多个文献**：用分号隔开；中外文混排时结尾句号用最后文献的语种。

**同一文献多次出现**
- 第一次必须引用完整信息；再次引用可略写为「前注○X，作者书/文，第X页」。
- 作者三人以上可只写第一作者加「等」。
- ⚠️ **投稿时不采用略写**（防编辑错乱），AI 终稿默认写全信息。

### 2.2 纸质文献

**信息要素**：主要作者 + 文献名称 + 其他贡献者（译/校）+ 出版信息（机构/年份或期刊期数）+ 页码。

**作者**
- 编辑作品加「主编/编/编著/译注」；只写主编，不写副主编。
- 外籍作者姓名前用方括号注国籍（如「[德]」「[美]」），以版权页为准。
- 译者位于文献名称**之后**、出版信息**之前**。
- 合作作者间用顿号；两人全列，三人及以上第一次全列，再引可只列首作者加「等」。

**文献名称**：用书名号《》、不用引号；有副标题的，主副标题间符号从原文。

**出版信息**
- 出版社前**不写城市**（只写「法律出版社」，不写「北京：法律出版社」）。
- 期刊写「《XX》YYYY年第X期」；社科版等用括号置于书名号内：`《清华大学学报（哲学社会科学版）》`。
- 报纸标注年月日及版次。
- 台湾地区文献遵守一个中国原则，不用「国立」「中央」；`《台大法学论丛》`不写「国立」。

**页码**：置于「第」「页」之间，只写一次「第」「页」；多页用「、」，连续页用「-」。如：`第55、64-68页`。

**示例**
```
[德]哈特穆特·毛雷尔：《行政法总论》，高家伟译，法律出版社2000年版，第55、64-68页。
陈兴良主编：《刑法学》，复旦大学出版社2003年版。
魏振瀛主编：《民法》（第7版），北京大学出版社、高等教育出版社2017年版。
瞿同祖：《中国法律与中国社会》，商务印书馆2010年版，第5-30页。
```

### 2.3 网络与自媒体

**网络文献**：引领词/作者/文章名后，标明网站名、网址、最后访问日期。
```
汪波：《哈尔滨市政法机关正对"宝马案"认真调查复查》，载人民网，
http://www.people.com.cn/GB/shehui/1062/2289764.html，2004年1月15日访问。
```

**微信公众号**：谨慎引用，限公信力强的官方公号；标明公号名与上传日期，**不建议标注链接**。
```
赵耀彤：《一名基层法官眼里好律师的样子》，载微信公众号"中国法律评论"，2018年12月1日上传。
```

**学位论文**：标明毕业学术单位与年份；已公开发表的改引发表后文献。
```
李松锋：《游走在上帝与凯撒之间：美国宪法第一修正案中的政教关系研究》，
中国政法大学2015年博士学位论文。
```

### 2.4 法律文件

**名称**
- 加书名号。「试行」「草案」「修正案序号」括注于书名号内：《刑法修正案（十）》。
- 不引起误解时「中华人民共和国」可省；《治安管理处罚法》。

**版本**
- 经修改的注明修正年份：《国家赔偿法》（2010年修正）第35条。
- 已失效的注明：「已废止」。

**条款序数**
- 行文中条、款、项序数用阿拉伯数字，序数中括号省略：《民法总则》第27条第2款第3项。
- 原文引用时从原文（条款用汉字，项用汉字加括号）：`第二十七条第二款第（三）项`。
- ⚠️ 法律文件**名称中**的条款序数不得改为阿拉伯数字。

**引用层级**
- 法律条文：一般只写名称+条文序数，不写通过日期/主席令等：《行政处罚法》第32条。
- 规范性文件：标明制定机关+文件号，文件号年份用六角括号〔 〕：
  `《国务院关于在全国建立农村最低生活保障制度的通知》（国发〔2007〕19号）`。

**外国法律/国际公约**：视情况加国别与年份；外文中文版可括注外文名。

### 2.5 司法案例

**基本格式**：标明案例来源（法院+文书名称+案号）。
```
浙江省高级人民法院（1988）浙法民上7号民事判决书。
北京市海淀区人民法院（1998）海行初142号行政判决书。
（2018）京0108民初142号民事判决书。
```

**案例名称**（正文）
- 民事/行政：「XXX（原告）诉XXX（被告）XXX（案由）案」，加引号：「刘燕文诉北京大学学位评定委员会案」。
- 刑事：「XXX（被告人）XXX（罪名）案」。
- 简称用原告/被告姓名：「刘燕文案」；避免误导性非正式名称。

**案号**：年份加圆括号；法院代码开头「0」不省，「字」字省略。

**指导性案例**：`最高人民法院指导案例24号（2014年）`。

**公报案例**：可只援引《公报》：`参见《最高人民法院公报》2015年第11期，第43-48页`。

## 三、与 md2word 预设的映射

| 本规范项 | md2word 配置位置 |
|----------|------------------|
| 正文四号仿宋 | `fonts.default`（name/size） |
| 题目二号宋体 | `titles.level1`（font/size） |
| 章节标题分级 | `titles.level2/3/4` |
| 行距/首行缩进 | `paragraph` |
| 页边距 | `page` |
| 页码 | `page_number` |

预设文件：`md2word/assets/presets/paper-os.yaml`

### 终稿导出命令

```bash
# 在 paper-os 终稿环节，默认使用 paper-os 预设
python ~/.claude/skills/md2word/scripts/md2word.py 终稿.md --preset=paper-os 终稿.docx
```

`/导出Word` 命令默认套用 `paper-os` 预设，一键输出符合本规范的 Word。

### 字体可用性

macOS 默认有「宋体-简」「楷体-简」「黑体」；「仿宋」「仿宋_GB2312」可能缺失，导出后 Word 会回退为相近字体。若目标机器为 Windows，宋体(SimSun)/黑体(SimHei)/仿宋(GB2312) 均自带。导出后请用最终提交环境核对字体回退情况。

## 四、调整入口

格式参数变更（如目标期刊要求不同字号/行距）：
1. **微调单次**：`--config=自定义.yaml`（基于 `paper-os.yaml` 改）
2. **固化默认**：直接编辑 `md2word/assets/presets/paper-os.yaml`
3. 同步更新本文件第 1 节的表格，保持「规范文档」与「预设」一致
