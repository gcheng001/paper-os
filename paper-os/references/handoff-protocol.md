# 论文写作OS 交接契约协议

> 每个步骤完成后，必须输出标准交接包，确保下游步骤能无缝接续。

---

## 交接包结构

每个步骤的产物文件头部必须包含 YAML frontmatter：

```yaml
---
step_id: S<N>
status: pending_review | reported | confirmed
key_conclusions:
  - 结论1（≤3条，一句话概括）
depends_on:
  - S<前置步骤>
deliverables:
  - 文件路径1
  - 文件路径2
---
```

### 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `step_id` | 是 | 步骤编号（S1-S8） |
| `status` | 是 | `pending_review`（硬门禁）/ `reported`（软确认）/ `confirmed`（用户已确认） |
| `key_conclusions` | 是 | 本步骤核心结论，≤3条，每条一句话 |
| `depends_on` | 是 | 前置步骤编号列表 |
| `deliverables` | 否 | 本步骤产出的文件路径列表 |

## 各步骤交接契约

### S1 → S2（选题 → 文献）

**S1 产出**：
- `outputs/S1-选题/选题报告.md`

**S2 读取**：
- 选题报告中的「研究问题」「认知差类型」「新颖性论证」
- 用于指导文献筛选方向和关键词

**交接要点**：
```
S2 需要知道：
├── 研究问题是什么（一句话）
├── 认知差类型（增加/删除/修改）
├── 关键词列表（中英文）
└── 预期读者画像（本领域学者）
```

### S2 → S3（文献 → 写作）

**S2 产出**：
- `outputs/S2-文献/文献矩阵.md`
- `outputs/S2-文献/文献笔记/*.md`

**S3 读取**：
- 文献矩阵中的「核心观点」「与本研究关系」「优先级」
- 高优先级文献笔记的「可引用点」

**交接要点**：
```
S3 需要知道：
├── 研究空白是什么（文献矩阵总结行）
├── 高优先级文献的核心观点（Top 5）
├── 可直接引用的关键论断
└── 文献中的争议点（用于讨论部分）
```

### S3 → S4（写作 → 讨论结论）

**S3 产出**：
- `outputs/S3-写作/写作蓝图.md`
- `outputs/S3-写作/初稿章节/*.md`

**S4 读取**：
- 写作蓝图中的「认知差贡献」「读者价值」
- 初稿章节中的「结果」部分

**交接要点**：
```
S4 需要知道：
├── 各章节的认知差贡献（蓝图）
├── 研究结果的核心发现（初稿·结果章）
├── 与常识相悖的结果（需要重点讨论）
└── 预期字数和结构要求
```

**回写规则**：
- S4 完成后独立保存为 `outputs/S4-讨论结论/讨论与结论.md`
- **不回写** S3 的初稿文件
- 如需修改 S3 内容，在 S4 产物中附「修改建议」附录

### S3+S4 → S5（初稿 → 自检）

**S5 读取**：
- 全文初稿（S3 各章节 + S4 讨论结论）
- 从结论倒查，不从引言顺读

**交接要点**：
```
S5 需要知道：
├── 完整论文文本（S3+S4 产物拼接）
├── 结论部分（逆序检查起点）
├── 研究方法说明（检查适用前提）
└── 数据来源说明（检查数据质量）
```

### S5 → S6（自检 → 核查）

**S5 产出**：
- `outputs/S5-自检/七问自检报告.md`

**S6 读取**：
- 七问自检报告中的「不通过项」
- 重点关注：引用准确性、数据来源、因果断言

**交接要点**：
```
S6 需要知道：
├── 全文文本（用于拆解事实声明）
├── S5 不通过项（重点核查方向）
├── 参考文献列表（逐条核查）
└── 法条/案例引用列表（元典核查）
```

### S6 → S7（核查 → PPT）

**S6 产出**：
- `outputs/S6-核查/事实核查报告.md`

**S7 读取**：
- 核查通过后的终稿
- 关键结论和发现

**交接要点**：
```
S7 需要知道：
├── 论文题目和研究问题
├── 核心发现（讨论部分精华）
├── 研究局限（诚实呈现）
└── 方法和数据概要
```

### S7 → S8（PPT → 答辩）

**S7 产出**：
- `outputs/S7-答辩PPT/PPT大纲.md`
- `outputs/S7-答辩PPT/slides.md`

**S8 读取**：
- PPT大纲（了解答辩结构）
- S5 七问自检报告（优先从薄弱点提问）

**交接要点**：
```
S8 需要知道：
├── PPT大纲（答辩结构）
├── S5 薄弱环节（模拟提问重点）
├── 研究方法细节（可能被追问）
└── 文献争议点（可能被问到不同观点）
```

## 交接异常处理

| 异常 | 处理方式 |
|------|----------|
| 前置步骤产物缺失 | 提示用户，允许跳步但输出警告 |
| 前置步骤 status 未确认 | 硬门禁步骤必须先确认，软确认步骤可继续 |
| 产物文件损坏/为空 | 提示用户重新执行前置步骤 |
| 用户自带论文直接进 S5+ | 独立模式：向用户索要必要信息，不建状态文件 |

---

## 独立模式交接

当子 Skill 被直接调用且无论文项目时：

1. 向用户索要必要输入（不依赖 state.json）
2. 产物存用户指定位置（默认桌面）
3. 不创建/更新 state.json 和 LOG.md
4. 产物 frontmatter 中 `depends_on` 为空数组
