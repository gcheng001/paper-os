# 论文写作OS 流程定义

## 主流程（8步）

```
S1 选题 → S2 文献 → S3 写作 → S4 讨论结论 → S5 自检 → S6 核查 → S7 PPT → S8 答辩
```

## 各步骤详细定义

### S1 paper-s1-topic（选题全流程）

| 项 | 内容 |
|---|------|
| 前置 | 无 |
| 门禁 | **硬** — 选题报告确认后才进S2 |
| 核心方法 | 五种选题导向 → 直接法/间接法细化 → 四项验证 → AI辅助流程 → 导师沟通模板 |
| 输出 | `outputs/S1-选题/选题报告.md` |
| 后置 | 更新 state.json + LOG.md + PAPER.md（题目与认知差字段） |
| 可选集成 | socratic-coach（选题模糊时）、元典（类案趋势/新颖性） |

### S2 paper-s2-literature（高效阅读文献）

| 项 | 内容 |
|---|------|
| 前置 | S1确认（可跳） |
| 门禁 | 软 |
| 核心方法 | 文献筛选优先级 → 卡尔非线性阅读顺序 → 笔记法 → 工具链（Zotero/VOSviewer/Elicit/Explainpaper） |
| 输出 | `outputs/S2-文献/文献矩阵.md` + `outputs/S2-文献/文献笔记/*.md` |
| 后置 | 更新 state.json + LOG.md |
| 可选集成 | 元典法条沿革+权威案例裁判要旨（法律文献补充） |

### S3 paper-s3-writing（论文写作构思与起草）

| 项 | 内容 |
|---|------|
| 前置 | S1确认（起草动作硬性要求，蓝图可先行） |
| 门禁 | **蓝图硬门禁**（确认后才起草），章节初稿软确认 |
| 核心方法 | 认知差三类型定位 → 读者画像 → 价值判断 → 卡片笔记写作法 → 分节起草 |
| 输出 | `outputs/S3-写作/写作蓝图.md`（硬门禁） + `outputs/S3-写作/初稿章节/*.md` |
| 后置 | 更新 state.json + LOG.md；章节产出**强制 de-ai-polish** |

### S4 paper-s4-discussion（讨论与结论专项）

| 项 | 内容 |
|---|------|
| 前置 | S3章节初稿存在（可跳：用户可拿现成初稿） |
| 门禁 | 软 |
| 核心方法 | 消除"So what"质疑 → 讨论3要素（发现/理论/实践）→ 首尾呼应 → 对象感 → 局限写法 → 对策建议3原则 |
| 输出 | `outputs/S4-讨论结论/讨论与结论.md` |
| 后置 | 更新 state.json + LOG.md；**强制 de-ai-polish** |

### S5 paper-s5-checklist（七问逆序自检）

| 项 | 内容 |
|---|------|
| 前置 | 全文初稿（S3+S4 或用户自带） |
| 门禁 | **硬** — 七项全过或用户明示接受残留问题 |
| 核心方法 | 逆序检查法（从结论倒查）：①结论呼应题目 ②结论有价值 ③局限有可做未做 ④结果与结论实质关联 ⑤充分讨论 ⑥分析过程错误 ⑦数据采集靠谱 |
| 输出 | `outputs/S5-自检/七问自检报告.md` |
| 后置 | 更新 state.json + LOG.md；不通过项须本步修改或用户明示接受 |

### S6 paper-s6-factcheck（AI事实核查）

| 项 | 内容 |
|---|------|
| 前置 | S5通过（可跳） |
| 门禁 | **硬** — 每条△/×须用户裁决 |
| 核心方法 | 拆解独立事实声明 → 逐项核查 → √/△/× 三级判定 → 总结统计 + 可信度评估 |
| 输出 | `outputs/S6-核查/事实核查报告.md` |
| 后置 | 更新 state.json + LOG.md |
| 核查通道 | 法条→元典 ft_detail 核对原文；案例→yuandian-case 按案号核对；一般文献/数据→WebSearch |

### S7 paper-s7-ppt（答辩PPT制作）

| 项 | 内容 |
|---|------|
| 前置 | S5/S6通过（可跳） |
| 门禁 | **硬** — 大纲确认后才生成slides |
| 核心方法 | 3环节模型（锁定客户/痛点/信号）→ 6部分框架（问题/价值/必要性/设计/讨论/限制）→ 每页<20字 |
| 输出 | `outputs/S7-答辩PPT/PPT大纲.md`（硬门禁） + `outputs/S7-答辩PPT/slides.md` |
| 后置 | 更新 state.json + LOG.md |
| 出口 | Gamma导入 / frontend-slides / guizang-ppt-skill 三选一 |

### S8 paper-s8-defense（答辩表现辅导）

| 项 | 内容 |
|---|------|
| 前置 | S7（可跳） |
| 门禁 | 软 |
| 核心方法 | 答辩≠辩论心态定位 → 5技巧 → 模拟答辩（AI扮演答辩委员提问） |
| 输出 | `outputs/S8-答辩/答辩准备清单.md` + `outputs/S8-答辩/模拟答辩记录.md` |
| 后置 | 更新 state.json + LOG.md |

## 交接约定

产物文件本身即交接包。每个产物头部带 YAML frontmatter：

```yaml
---
step_id: S<N>
status: pending_review | reported
key_conclusions:
  - 结论1（≤3条）
depends_on:
  - S<前置步骤>
---
```

下游步骤读取上游产物的 frontmatter 获取关键结论，读正文获取详细内容。
