# S6b 专业评价（顶刊同行评议）

## 工作定位

| 项 | 内容 |
|---|------|
| **前置条件** | S5通过（可跳） |
| **门禁类型** | **软确认 reported** — 汇报评议结果即可，不阻断后续 |
| **阶段定位** | **可选阶段** — 实证/理论性论文建议执行；纯规范/案例类可跳过 |
| **后置动作** | 更新 state.json（S6b→reported）+ 追加 LOG.md |
| **de-ai-polish** | 不强制（本步是评议，不是写作） |

---

## 核心理念

### 与 S6 事实核查的关系

| 维度 | S6 事实核查 | S6b 专业评价 |
|------|-----------|------------|
| **管什么** | 真/假（技术性硬错误） | 价值与逻辑（学术贡献度、研究定位、建设性改进） |
| **核查对象** | 文献是否真实、数据是否准确、法条是否现行 | 研究是否重要、方法是否最优、结论是否站得住 |
| **产出** | 独立事实核查报告 | 独立专业评价报告 |

两者独立产出，互不替代。建议顺序：S5 → S6 → S6b（先排除硬错误，再做价值判断）。

### 与 S5 七问自检第⑥问的去重

- **S5 第⑥问**："分析过程有错误吗？" — 作者**自查**逻辑
- **S6b**：以**外部审稿人**视角再审一遍

**强制去重协议**：S6b 执行前**必须先读取** `outputs/S5-自检/七问自检报告.md`，重点复核 S5 标记的薄弱项是否被外部视角推翻，**不重复检查 S5 已通过项**。

### AI 评议的定位

- AI 可参考但**不可全信**
- 拿不准的与导师商讨
- 核心价值：发现盲区、提供改进思路，而不是替作者做学术判断

## 执行流程

> 完整提示词模板见 `references/professional-review.md`

### Step 1：全局扫描定位

- 通读论文，判断**核心问题**（作者到底在解决什么）
- 明确研究在**领域研究图谱**中的位置
- 核验作者"首次"声称是否属实（"国内首次""尚无研究"等断言）
- 对标近 2 年前沿，列 2-3 篇最相关高质量文献带链接

### Step 2：深度逻辑剖析

- 审查**研究逻辑链条**每一环是否必然成立
- 诊断**方法是否最优**（有没有更合适的工具被忽略了）
- 挖掘**亮点**（哪怕有缺陷的研究，也要找到值得肯定的创新点）

### Step 3：建设性反馈（三明治结构）

| 层 | 内容 |
|---|------|
| 开篇 | 概括核心贡献（一句话讲清作者做了什么、为什么重要） |
| 中间 | 分级问题诊断（关键缺陷 + 重要疑虑 + 改进建议） |
| 结尾 | 指明修改后的发表潜力（鼓励 + 方向） |

## 贡献度四级判定

必须对论文做出**明确归类**并给出判定依据：

| 级别 | 判定标准 |
|------|---------|
| **开创性** | 开辟新领域/新范式，提出前人未触及的关键问题，方法或理论具颠覆性 |
| **重要推进** | 在已有方向上做出实质性突破，解决长期悬而未决的问题，或显著提升已有方法 |
| **增量贡献** | 对已有研究的补充、完善、拓展，方法或结论有新意但非颠覆 |
| **有限价值** | 重复已有工作、贡献微小，或方法/结论存在难以弥补的根本缺陷 |

## 输出约定

保存到：`outputs/S6b-专业评价/专业评价报告.md`

产物 frontmatter：

```yaml
---
step_id: S6b
status: reported
key_conclusions:
  - 贡献度级别：开创性/重要推进/增量贡献/有限价值
  - 最关键缺陷：(1条)
  - 核心修订建议：(1条)
depends_on:
  - S5
deliverables:
  - 详细问题诊断与修订路线图见正文（关键缺陷/重要疑虑/改进建议分级清单 + 给主编机密评语）
---
```

> **key_conclusions 严格 ≤3 条**（handoff schema 约束 maxItems:3）。
> 详细问题、分级诊断、修订路线图、给主编机密评语一律放**正文**和 deliverables 说明里，不要塞进 key_conclusions。

### 独立模式

用户可单跑专业评价，提供任意论文文稿即可：
- 产出存桌面（默认 `~/Desktop/专业评价报告_<论文短名>.md`）
- **不建 state.json**，不进入论文OS状态机
