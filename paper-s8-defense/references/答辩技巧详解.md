## 一、答辩的重要性

### 三个核心认知

1. **论文真伪验证**：答辩是确认论文确实出自你手的环节。如果无法在答辩时顺利回答老师的问题，别人会怀疑论文是否代笔。
2. **工匠精神信号**：在低激励环境下（毕业论文已写完、成绩差别不大），你的表现传递了态度信号。企业招聘时若善于利用这个标准，可以更有效地筛选人才。
3. **数据社会风险**：行为痕迹会计入信用评估，毕业论文成绩在未来可能成为重要的评估维度。

### 关键认知纠正：答辩 ≠ 辩论

| 辩论 | 答辩 |
|------|------|
| 双方对抗 | 老师帮你发现问题 |
| 赢为目标 | 改进为目标 |
| 坚持立场 | 虚心接纳 |

## 二、PPT制作的3个环节（产品设计思维）

### 环节1：锁定客户

客户 = 答辩组老师（专业人士，认知负荷高）。

**删除一切基础科普内容**：
- ❌ "SWOT是指……它分别对……进行分析"
- ❌ "根据第××次全国互联网……报告的数据，我国已有网民……"
- 这些内容答辩组老师比你清楚得多，反复唠叨是一种冒犯

### 环节2：了解痛点

答辩组老师的痛点：学生多、水平参差、PPT质量低、认知负荷重。

**解决方案**：
1. **严格控制文字量**：每页汉字字符或英文单词 ≤ 20 个
2. **用美观正确的图表**：丑图或错误的图会让印象大打折扣
3. **脱稿讲述 + 目光接触**：老师们会逐渐抬头看你，目光变得温和

**脱稿技巧**：
- 一开始做不到从容脱稿 → 事先写好稿子多读几遍
- 实在背不下来的地方写在小卡片上当提词器
- ❌ 千万不要从头到尾低头念卡片

### 环节3：有效传递信号

突出展示6个部分（共6-7页）：

| 页 | 内容 | 要点 |
|----|------|------|
| 1. 问题 | 一句话说清研究问题 | 不用从句和被动句式，越简明越好 |
| 2. 价值 | 解决问题有什么用 | 不随意拔高，实事求是 |
| 3. 必要 | VOSviewer/CiteSpace文献分析图 | 说明研究处于哪个夹缝，别人没解决的 |
| 4. 设计 | 数据收集/处理/分析 | 简要说明 |
| 5. 讨论 | 核心部分 | 剖析违背常识的结果，给出解释 |
| 6. 限制 | 研究的不足 | 实事求是，有一说一 |

## 三、AI辅助做PPT

### 生成讲解内容提示词

```
请查询最新资料，用小学生也能听懂的语言，详细、生动地讲解以下内容：
[论文各部分的内容概要]
```

### 转换为slides提示词

```
请将以下内容转换为 Markdown revealjs slide 格式。
要求：每段对应一页slide，每页不超过6项内容，数学公式用LaTeX。
```

### Gamma导入流程

1. 将大纲转换为 Markdown revealjs slide 格式
2. 导入 Gamma（Text transform模式）
3. 设置：简体中文 + 12张上限 + 自动配图

## 四、答辩五技巧

### 技巧1：PPT不过长，重点突出

帮助老师快速聚焦。6-7页足矣。

### 技巧2：脱稿讲述，抓住重点

- 不要逐字念PPT
- 用自己的话讲清楚每页的核心
- 目光接触答辩委员

### 技巧3：拿笔记本和笔，充分准备

- **你要对自己的学位负责**
- 答辩秘书的笔记 ≠ 你的记录
- 你负责验证和回应

### 技巧4：认真记录老师的问题

- 全面且准确地记录
- 不要遗漏任何问题
- 记录后逐一回答

### 技巧5：回答以开放、包容、谦虚的心态

- "有则改之，无则加勉"
- 有保留意见 → 会后与导师沟通（不要当场争辩）
- 老师的目的是帮你，不是刁难你

## 五、模拟答辩策略

### 提问优先级

| 优先级 | 提问来源 | 示例 |
|--------|---------|------|
| 最高 | S5不通过或勉强通过的项 | "你的结论和题目的关联性不够强，能解释一下吗？" |
| 高 | 方法论薄弱点 | "你的样本代表性如何保证？" |
| 中 | 文献空白 | "你怎么看XX学者的不同观点？" |
| 一般 | 延伸问题 | "如果扩展到其他领域，你的结论还成立吗？" |

### 常见问题类型

1. **选题类**：为什么选这个题目？有什么价值？
2. **方法类**：为什么用这个方法？不用其他方法？
3. **数据类**：样本量够吗？代表性如何？
4. **结论类**：你的结论有什么新意？局限是什么？
5. **延伸类**：如果重做一次，你会怎么改进？

### 回答模板

```
感谢老师的提问。关于[问题]，我的理解是……

[正面回答]

当然，这个问题确实存在[局限/不足]，如果进一步研究，我会[改进方向]。
```
