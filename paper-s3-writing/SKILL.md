## 执行流程

### Step 1：认知差定位

论文价值的唯一衡量标准 = **认知差**（读者读前读后认知的变化）。

三种类型：

| 类型 | 定义 | 示例 |
|------|------|------|
| 增加型 | 读者不知道 → 知道了 | 首次梳理某新兴领域法律框架 |
| 删除型 | 读者信错 → 被纠正 | 用实证推翻主流认知 |
| 修改型 | 读者认知迭代演进 | 证明适用条件比通说更窄/更宽 |

**验证两个问题**：
1. 选题能否让读者产生认知差？
2. 读者是否对这个选题感兴趣？（文献热度验证）

### Step 2：读者定位

**正确定位**：专业领域学者（你所在系的其他老师）

**错误定位**：
- ❌ 普罗大众（不需要做科普）
- ❌ 同学（他们不是论文的目标读者）
- ❌ 导师（他是顾问/评审人，不是普通读者）

**对象感检验**：这句话能否删除而不影响意思表达？能删→删，提高信息密度。

### Step 3：生成写作蓝图（硬门禁）

为论文每个章节标注其认知差贡献和读者价值：

```markdown
# 写作蓝图

## 论文结构
| 章节 | 内容概要 | 认知差贡献 | 读者价值 |
|------|---------|-----------|----------|
| 引言 | ... | 引出问题 | 建立研究必要性 |
| 文献综述 | ... | 梳理现状 | 展示研究空白 |
| 研究设计 | ... | 方法说明 | 可复现性 |
| 结果 | ... | 数据呈现 | 证据支撑 |
| 讨论 | ... | 深度阐释 | 消除"So what" |
| 结论 | ... | 首尾呼应 | 认知差交付 |

## 各章节预计字数
## 写作顺序建议
```

**→ 用户确认蓝图后才进入起草**

### Step 4：分节起草

#### 核心原则：快写慢改

- **快写**：用最短时间形成完整作品原型。一气呵成，不要边写边编辑。
- **慢改**：在原型基础上不断迭代改进，提升质量。

**为什么不能边写边编辑？**
1. 阻断思路——奇思妙想转瞬即逝
2. "沉没成本"效应——精心编辑的文字，即使该删也舍不得
3. 消耗时间——编辑动作吃掉本应用于创作的时间

#### 卡尔教授8步写作顺序（非线性！）

```
① 列出大纲 → 概览全貌，不迷失方向
② 展示图表（研究结果）→ 只放图表，不写描述
③ 写实验部分 → 数据来源、方法、流程、结果
④ 写讨论部分 → 自问自答消除读者疑惑
⑤ 写结论 → 发现+局限+展望三段式
⑥ 写摘要（含关键词）→ 要言简意赅，全面反映特点
⑦ 写文献回顾 → 用卡片笔记拼接文献网络
⑧ 写引言 → 最后写，最难的部分放最后
```

**"拼图"写作法**：不按顺序写，而是依照大纲指引，不断选择最容易切入的地方填入组块内容，最后"拼装"出整篇论文。

#### 卡片笔记写作法（对抗拖延症）

**原理**：纳博科夫的创作秘诀——在卡片上写下各种文字，用它们拼出整张拼图。卢曼用此方法30年出版58部著作、数以百计论文。

**为什么有效？**
- 单张卡片容量有限→任务变简单→激发行动力
- 每张卡片是一个组块→几张卡片=一个小节→几小节=一章→整部著作

**工作流程**：
1. **输入**：语音输入/手写涂鸦→快速记录闪念笔记
2. **组织**：在Heptabase/Obsidian等工具中建立卡片→白板排布→连线确定顺序
3. **输出**：按连线顺序拷贝到编辑工具→添加承上启下连接→补充缺失内容→格式调整润色

**占位符标注法**：遇到需要引用的地方直接标记"..."（三个句号），同时把想表达的意思记在一旁，迅速回到写作流程。完成初稿后统一查找补全。

#### 开头原则——开门见山

- ❌ 俗套路径：宏观数据 → 行业现状 → 逐渐逼近问题
- ✅ 推荐路径：问题 → 价值 → 必要性 → 设计 → 结果

#### 写作障碍应对

1. **创造无干扰环境**：降噪耳机、物理隔离区
2. **避免一边写作一边研究**：遇到卡壳问题用AI（Perplexity+R1）即时解决
3. **设置专属写作时段**：每天固定2小时纯写作时间，不查阅新文献
4. **占位符标注法**：标记"..."留到后期统一处理

#### 每节起草后

1. **强制调用 de-ai-polish**（去AI味）
2. 软确认入库
3. 保存到 `outputs/S3-写作/初稿章节/<章节名>.md`

### Step 5：AI辅助研究与文献回顾

#### 卡壳时用AI即时解决

遇到写作中卡壳的研究问题，用Perplexity（选择Reasoning with R1模式）即时解决：

**增强提示词（生成可直接插入文章的"卡片"）**：

```
查看用户输入，可能为论点或者问题，你需要对它进行补充或回答。

流程：
如果是论点，尝试经过深思熟虑，用合理的逻辑加以论证，并且搜索相关证据加以支撑。
如果是问题，则尝试给出你的回答，然后将其当成论点，进行论证和支撑。

当你有了一个完整的论证过程（包含论点、论证和论据）时，则尝试提出可能的疑问，
然后对疑问进行驳斥，并且对自己的驳斥给出完整的论证过程和证据。

注意：
- 越详细越好
- 每个论证过程给出2个或以上证据，避免孤证
- 证据搜索用英文，来源优先级：peer review paper > conference paper > 权威媒体 > preprint
- 搜索注意时效性，优先使用更新的材料
- 获取搜索结果后用常识反思，过滤错误内容
- 每条证据后用Markdown内嵌链接给出原始文献url
- 用简体中文输出
```

#### AI辅助文献回顾提示词

```
查看用户输入，检索高质量英文学术文献，写成一篇简体中文深度研究报告，越详细越好。

对每一篇引用到的文献，除了详细叙述其贡献、特点，还要根据实际情况做出点评，
并且指出research gap（研究空白）。

要求使用自然语言段落，而不是列表形式呈现。
引用论文来源要准确，使用APA格式，用Markdown内嵌链接给出对应原始文献的url。
搜索注意时效性，优先使用更新的材料。
获取搜索结果后用常识反思，过滤错误内容。

用简体中文输出。
当前日期：{{填写当前日期}}

下面是用户输入：
```

> **警告**：AI生成的文献回顾必须逐条核对引用来源链接，确认文献真实存在。

### Step 6：样例参照

起草每个标准章节时参照样例解析中对应注解。
> 完整样例见 `references/样例解析.md`（问卷调查+卡方检验论文逐节注解，注1-注34）

**样例关键注解速查**：
- 注1：题目要直观体现核心问题
- 注6-7：宏观数据要用于对比，不要堆砌
- 注8：引言中的论断必须有文献支撑
- 注11：说"尚未发现"要谦逊留余地
- 注14：研究设计才是展现功力的舞台
- 注22-23：原假设vs备择假设的正确用法
- 注25：统计推断不能宣称"证明"
- 注26：讨论=消除读者疑惑的自问自答
- 注27：相关分析≠因果分析，不能推导因果建议
- 注28-31：结论=发现+局限+展望三段式


写作蓝图 frontmatter：
```yaml
---
step_id: S3
status: pending_review
key_conclusions:
  - 论文结构：X章
  - 认知差类型：增加/删除/修改
  - 核心章节：（最重要的2个）
depends_on:
  - S1
---
```
