# S1 论文选题全流程

## 工作定位

| 项 | 内容 |
|---|------|
| **前置条件** | 无 |
| **门禁类型** | **硬门禁** — 选题报告须用户确认后才能进入S2 |
| **后置动作** | 更新 state.json（S1→pending_review）+ 追加 LOG.md + 更新 PAPER.md（题目/认知差） |
| **de-ai-polish** | 建议（选题报告） |

---

## 适用场景

法律实务人员（律师、法务、检察官、法官等）需要撰写法律研究论文、实务文章或学术论文时，辅助完成选题论证。通用学术论文亦可适用（法律检索部分跳过）。

## 核心理念

**选题决定论文价值的一半。** 好的选题应当：
- 有认知差（增加/删除/修改读者原有认知）
- 有读者感兴趣
- 可执行（在能力和资源范围内）

**先避开一个坑：方法导向选题**
> 不要因为学了某个时髦方法就硬凑选题。方法必须匹配问题，而不是反过来。

**方法导向的典型陷阱：**
- 学了爬虫 → 硬凑数据采集选题
- 学了深度学习 → 用DL做表格数据分类，结果还不如逻辑回归
- 学了R语言 → 堆砌漂亮图表但研究问题空洞

**选择方法前必问三个问题：**
1. 为什么要用这种方法？是因为自己熟悉，还是因为别人不熟悉？
2. 这个问题是否适用于这种方法？有没有更好的替代？
3. 能获取这种方法需要的数据吗？数据经处理后能否得到需要的结果？

## 第一阶段：选题方向探索

### 五种选题导向

#### 1. 文献导向

**方法A：文献冲突法（吴东军教授法）**
找两篇观点"打架"的文献。文献A提出某观点，文献B用充足证据驳斥。但文献A通过了审稿，很可能有合理之处。你的研究就是找到更全面的框架，在何种条件下文献A依然成立。

**方法B：瓦里安法（先做后读）**
不要先读文献，先独立构建自己的分析框架，迭代修正后再去读文献。如果你的框架和前人有细微差别，从中可能有极其有价值的发现。

**方法C：展望追踪法**
阅读文献的"不足与展望"部分。但注意：大部分能解决的问题原作者早已解决，留下的要么极其困难，要么工作量足以单独成文。

#### 2. 数据导向

从独特数据中发现研究问题。

**法律实务中的数据来源：**
- 元典案例数据库（裁判文书、法规、企业信息）
- 企业工商/涉诉数据（通过元典企业信息接口获取）
- 自己办理案件的实务数据
- 行业协会/主管部门公开数据

**关键原则：** 专属数据 > 开放数据；唯快不破；数据独特性决定研究价值。

#### 3. 实践导向

**方法A：实务痛点法** — 当事人反复问的问题？裁判结果不合理？"大家都这么做但没人研究"？

**方法B：案例研读法** — 用元典检索同类案件，发现裁判逻辑中的矛盾或空白。

**方法C：媒体观察法** — 关注深度报道，区分"流量话题"和"真问题"。

#### 4. 协同导向

**保尔·帕夫罗三原则：** 努力工作、问题要有趣、找靠谱合作者。

**法律实务合作模式：** 律师×学者、不同专业律师、律师×行业专家。

#### 5. 系统导向

在一个领域的每个方向上占领关键点。**众争勿往**——不要扎堆热门，关注被忽视的交叉地带。

## 第二阶段：选题细化

### 直接法：从自身经验出发

核心是**研究敏感性**——抓住问题的敏感度 + 立即深入的执行力。

**经典案例**：有位老师为了查文献熬夜，痛苦中冒出问题：别的研究者是不是也在熬夜？他没有放过这个念头，放下手头研究，真的去探究科研人员熬夜这件事。利用CNKI的用户行为日志（下载时间、时区等），发现熬夜现象普遍且规律性强，不同国家/地区差异大。进一步扩展到周末、午休时间的文献下载情况。这篇文章后来发表在本学科顶级期刊上。

**关键**：同样的场景，你能否凭借研究敏感性找出巧妙的选题？把"日用而不知"的事情变成有意识的观察和思考。

实务中的困惑、当事人反复追问的问题、行业"约定俗成"的做法，都可能是选题种子。

> **选题模糊时**：可选调用 `socratic-coach` 追问澄清，帮助理清研究问题的边界。

### 间接法：从阅读中获得启发

跨领域读书 + 边读边记笔记。

**两个关键点：**
1. **读的不只是本专业书**——跨领域阅读能带来独特视角。面对同样的问题，你的阅读积累会让你触类旁通。
2. **必须记笔记**——"笔记不是辅助思考的工具，笔记本身就是思考。"（费曼）好的笔记不仅帮助思考、激发灵感，还会极大减轻后续写作压力。

> 很多人通过阅读提升自我，在看待相同事物时获得与年龄不相称的成熟洞见。你看不到的痛点，他们可以轻易察觉。久而久之，选题时他们"信手拈来"，你只能围绕校园里几座建筑物绞尽脑汁。

## 第三阶段：选题验证

### 四个必检项（开题报告不通过的4个原因）

**1. 是否"出了圈儿"？**（最严重，直接"毙掉"）
- 跨学科研究必须体现本学科视角
- 例：信息管理专业学生改进了Linux内核→贡献再大也只能申请计算机学位
- **保证不出圈儿**：大量阅读本领域文献感知学科边界，或请导师把关
- 法律论文须落脚在法律规范或制度上

**2. 是否"没价值"？**
- 标准：除了你，至少还要有一个人真正关心研究结果
- 警惕：那些领先于时代、无人关心的选题→用贝叶斯公式想想你做出惊艳研究的可能性有多大
- 更多时候，不被别人看重的研究，是确实没意义甚至很无趣的

**3. 是否"不必要"？**
- 别人已经做完并发表了→开题小组组长问"某博士2015年那篇看了吗？"你答"没有"→重选
- **常见陷阱**：只看中文文献、只在CNKI用中文关键词搜→英文文献遗漏严重
- 必须全面读文献，英语文献阅读必不可少

**4. 是否"不靠谱"？**
- 题目过于宏大（"企业信息化发展策略研究"→几本大部头都做不完）
- 方法太复杂（同时集成结构方程+深度学习+复杂系统仿真→简历支撑不了）
- 数据难获取（网站不向你开放用户数据→别人能用是特殊机缘）
- **避免方法**：选题时往后多想一步，做小规模测试验证可行性

### 认知差检验

| 类型 | 含义 | 法律领域示例 |
|------|------|-------------|
| **增加型** | 不知道→知道 | 首次系统梳理某新兴领域法律规制框架 |
| **删除型** | 信错→纠正 | 用实证数据推翻主流认知 |
| **修改型** | 认知迭代 | 证明某制度的适用条件比通说更窄/更宽 |

**验证两个问题**：
1. 你的选题是否会让读者产生认知差？→ 检索文献确认无人提出过相同观点
2. 你的选题是否有读者感兴趣？→ 用VOSviewer/CiteSpace分析文献热点趋势

> **VOSviewer/CiteSpace**：可视化文献计量工具，可快速呈现某领域的主要话题、重要论文、高影响力学者及三者关联。科研入门者推荐先用VOSviewer（更简单易学），CiteSpace功能更强大但参数众多容易用错。

## 第四阶段：AI辅助选题

AI不能替你完成选题，但AI博览群书，可以高效拓展视野。

### 流程一：初筛相关领域

```
你是一名优秀的资深科研项目负责人（PI），完成过多项高水平基金项目。

我需要你根据我的研究问题，一步步思考，充分挖掘你的经验，用你最有信心的方式，给出咨询建议。

如果你要为这个研究问题组建团队，那么你需要招募的科研人员的具体研究方向应该是哪些？

对你给出的每一个研究方向，请详细说明它与本研究课题的关联，并且使用近期相关的高水平研究成果加以印证。

我要研究的问题是：{{研究主题}}
```

### 流程二：细化研究问题

```
你是一名{{研究方向}}领域的优秀科研专家。

根据用户提供的研究问题—{{研究主题}}，首先搜索你研究方向与本课题相关的近期高水平研究，并充分阅读。

之后执行以下步骤：

1. 根据你的经验，为自己撰写简要的专业人物小传。

2. 从你的独特视角出发，就我们的研究主题，提出3个高水平、新颖的研究问题。

3. 对每个问题：
   a. 确保该问题与你的研究方向一致，且不能与你能找到的任何近期高水平研究相重叠。
   b. 一步步思考，反思你提出问题的依据。
   c. 为提出问题的质量打分，要提供打分的分维度构成与合理性根据。
```

### 流程三：验证选题新颖性

```
你是一名优秀的参考咨询馆员，善于根据文献检索，智能评判用户选题的新颖性和必要性。

请你根据用户提供的研究题目，在高水平文献中找寻与之最为相关的篇目。然后仔细阅读，一步步思考，对每一篇文献进行以下操作：

1. 判定该文献和用户研究题目的重合程度，按照百分比打分，并且详细说明理由。

2. 如果重合程度不是100%，说明用户选题与该文献之间的差异，并且说明该差异的研究价值。

3. 最后，输出你综合评判该题目的新颖性评分，并且说明打分依据。

用户研究题目为{{选题}}。
```

**结果解读：** 60%以下→新颖性较高可推进；60-80%→看差异是否够一篇论文；80%+→调整角度或放弃。

### 流程四：确定门槛

兴趣（愿意花半年研究？）+ 能力（掌握所需方法？）+ 资源（数据可获取？）。最终剩2-3个题目，带着选题报告与导师讨论。

## 第五阶段：与导师/合作者沟通

### 三个常见误区（根因分析）

**误区1：把论文当作文**
- 写作文：自己有话可说、凑足字数就行
- 写论文：必须提供认知差，写给本领域专业人士看
- 导师拦你是"用垒鸡窝的经验去盖大楼"，及时拦下是为你好

**误区2：不验证选题就提交**
- 学生以为：我提选题，导师来验证
- 导师默认：你应该先做验证，他只检验你的验证过程是否合理
- 你交的是"半成品"，在导师眼里就是想偷懒

**误区3：不会沟通**
- 导师希望：看到你用合理方法选题并验证，他在此基础上提意见
- 错误做法：坚持预设、不找导师要反馈、忽略导师意见继续我行我素
- 正确做法：仔细询问和倾听导师意见，做下一步修订

### 沟通模板（必备要素）

1. **题目**：完整列出，字斟句酌
2. **认知差类型**：增加/删除/修改
3. **兴趣论证**：列出文献说明学术界对此主题的关注
4. **文献佐证**（根据认知差类型展开）
5. **研究方法**：与研究问题匹配
6. **疑虑与困难**：坦诚说明

## 输出约定

### 选题报告结构

保存到：`outputs/S1-选题/选题报告.md`

产物文件头部 frontmatter：
```yaml
---
step_id: S1
status: pending_review
key_conclusions:
  - （选题名称）
  - （认知差类型与说明）
  - （新颖性评估结论）
depends_on: []
---
```

正文结构：
```
一、选题名称
二、研究问题（一句话概括）
三、选题导向（五选一或组合）
四、认知差类型（增加/删除/修改，附说明）
五、价值论证（谁会关心这个结果？）
六、新颖性论证（与已有研究的差异，附文献对比）
七、可行性评估（方法/数据/时间）
八、潜在困难（坦诚说明，附应对思路）
```

### 独立模式

无论文项目时直接向用户索要研究方向，产出选题报告存桌面，不建状态文件。

## 注意事项

1. **人主导，AI辅助**：AI是初级合作者，不是决策者
2. **不要迷信评分**：核心是分析过程和差异发现
3. **全面检索**：不要只看中文文献，关注域外研究
4. **小题大做**：题目要具体，不要过于宏大
5. **先做后读**：瓦里安法——先独立思考，再参考文献
6. **交叉视野**：跨领域读书能带来独特视角
